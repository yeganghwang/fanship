## 일반 작업
*   나열된 모든 엔드포인트에 대해 일관된 접근 방식(예: Axios, Fetch API)을 사용하여 API 통합을 구현합니다.
*   인증을 위한 JWT 토큰 처리:
    *   토큰을 안전하게 저장합니다(예: HttpOnly 쿠키, 적절한 보안 고려 사항이 있는 localStorage).
    *   인증된 요청에 토큰을 자동으로 포함합니다.
    *   토큰 만료를 처리하는 로직을 구현합니다(예: 로그인으로 리디렉션, 메시지 표시).
*   저장된 JWT 토큰을 지워 클라이언트 측 로그아웃을 구현합니다.
*   `0.2. 데이터 검증 규칙`에 명시된 대로 사용자 입력(사용자 이름, 비밀번호, 이메일, 닉네임 등)에 대한 클라이언트 측 데이터 유효성 검사 규칙을 구현합니다.
*   모든 목록 보기 API에 대한 페이지네이션 UI 및 로직을 구현합니다:
    *   회사 목록 (`2.2`)
    *   회사별 셀럽 목록 (`2.3`)
    *   사용자 즐겨찾기 목록 (`3.3`)
    *   사용자 게시글 목록 (`3.4`)
    *   게시글 목록 (`4.2`)
    *   댓글 목록 (`4.7`)
    *   셀럽 스케줄 목록 (`5.2`)
    *   굿즈 목록 (`6.2`)
    *   사용자 굿즈 목록 (`6.6`)
*   프로필 이미지 업로드 기능(지원 형식, 최대 크기, 권장 치수 고려)을 구현합니다.
*   공통 오류 응답 형식(`400`, `401`, `403`, `404`, `409`, `500`)을 기반으로 강력한 오류 처리 및 사용자 친화적인 메시지 표시를 구현합니다.

## 인증 / 사용자 (섹션 1)
*   **1.1 회원가입:**
    *   `username`, `password`, `mail`, `nickname`, `position`, `dob`, `ig_url`, `pfp_img_url`, `company_id`, `celeb_type` 필드를 포함하는 회원가입 양식을 만듭니다.
    *   `/api/auth/register`로 POST 요청을 보냅니다.
    *   성공(예: 로그인으로 리디렉션) 및 오류 응답을 처리합니다.
*   **1.2 로그인:**
    *   `username`, `password` 필드를 포함하는 로그인 양식을 만듭니다.
    *   `/api/auth/login`으로 POST 요청을 보냅니다.
    *   수신된 `access_token`과 `user_id`를 저장합니다.
    *   성공 및 오류 응답을 처리합니다.
*   **1.3 로그아웃:**
    *   로그아웃 버튼/메커니즘을 구현합니다.
    *   JWT 토큰과 함께 `/api/auth/logout`으로 POST 요청을 보냅니다.
    *   클라이언트 측에 저장된 JWT 토큰을 지웁니다.
*   **1.4 사용자 프로필 조회:**
    *   사용자 프로필 표시 컴포넌트를 만듭니다.
    *   `/api/users/{user_id}`로 GET 요청을 보냅니다.
    *   사용자 세부 정보: `nickname`, `position`, `pfp_img_url`, `ig_url`을 표시합니다.
*   **1.5 사용자 정보 수정:**
    *   프로필 편집 양식을 만듭니다.
    *   JWT 토큰과 함께 `/api/users/{user_id}`로 PATCH 요청을 보냅니다.
    *   `nickname`, `password`, `pfp_img_url`, `ig_url` 업데이트를 허용합니다.
    *   성공 및 오류 응답을 처리합니다.
*   **1.7 비밀번호 재설정 요청:**
    *   `mail` 필드를 포함하는 "비밀번호 찾기" 양식을 만듭니다.
    *   `/api/auth/password-reset-request`로 POST 요청을 보냅니다.
    *   성공/오류 메시지를 표시합니다.
*   **1.8 비밀번호 재설정 확인:**
    *   `token` 및 `new_password` 필드를 포함하는 비밀번호 재설정 양식을 만듭니다.
    *   `/api/auth/password-reset-confirm`으로 POST 요청을 보냅니다.
    *   성공/오류 메시지를 표시합니다.
